androir-mcp

Drive real Android devices from any AI agent.

An independent Model Context Protocol server that lets an agent see the screen, read the live UI tree, and drive input — backed entirely by adb + uiautomator. No native code, no device-side app.

The tool set

A small, conventional automation surface

A screen-read + input tool set that drops into any MCP-based agent loop. Every tool takes an optional serial; coordinates are in device pixels, so describe_screen tap points feed straight into tap with no translation.

screenshot

Capture the device screen as a signature-validated PNG via exec-out screencap.

describe_screen

Parse the uiautomator UI tree into a flat list of labels with center tap coordinates — exact bounds and text, entity-decoded.

tap / swipe / long_press

Coordinate input in device pixels via input tap / swipe.

type_text

Type arbitrary text safely — spaces, metacharacters, and unicode all type verbatim, single-quoted for the device shell.

press_key / home / back

Named key events — home, back, enter, recents — mapped to keycodes.

launch_app / open_url

Launch by package or a friendly name (resolved from the installed list, cached per device); open an http(s) URL in the default browser.

list_targets / status

Enumerate connected devices and read device state, properties, and battery.

Safety by construction

argv-only adb, strict serial validation, per-call timeout with process-group kill, and scrubbed errors — no raw stderr or host stack traces reach the model.

Why androir

Real UI tree, not OCR

Everything is backed by adb and uiautomator: there is no native code and nothing to install on the device. It is its own project — not a copy or port of anything.

Exact bounds, not guesses

describe_screen returns the real UI tree straight from uiautomator — element bounds and text — so the agent taps real coordinates rather than guessing from a screenshot.

argv-safe by design

Every adb command is an argument array, never a shell string; values handed to the device shell are single-quoted for it. There is no shell-injection surface.

Local & credential-free

The server runs locally, holds no credentials, and talks only to the device(s) adb already sees. Errors are scrubbed before they reach the model.

Getting started

Install & attach

Requires adb (Android platform-tools) on your PATH and a device with USB debugging enabled (or an emulator). Confirm it's visible with adb devices.

1 · Build

Clone, install, and compile the TypeScript server.

# clone & build
git clone https://github.com/benasbarciauskas/androir-mcp.git
cd androir-mcp
npm install
npm run build

2 · Attach to an MCP client

Point your client at the built server in dist/index.js.

{
  "mcpServers": {
    "androir": {
      "command": "node",
      "args": ["/path/to/androir-mcp/dist/index.js"]
    }
  }
}
Verify it works. With a device connected, run npm run selfcheck — it lists targets, takes a screenshot, and dumps the UI tree, printing PASS when the three core tools work end-to-end against a real device.
